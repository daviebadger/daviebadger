**What** I use:

* **Terminal:** [Alacritty](https://github.com/alacritty/alacritty)

**Why** I use:

<details>
  <summary>Alacritty</summary>

  * **configuration**

    I want a configurable terminal emulator with configuration stored inside a user configuration directory and in a plain text file. Thus no dconf-based ones like [GNOME Terminal][GNOME Terminal] or [Tilix][Tilix].

  * **cross-platform**

    I want a platform-independent terminal emulator with support for multiple operating systems to avoid vendor lock-in. Thus no platform-specific ones like [iTerm2][iTerm2] or [Windows Terminal][Windows Terminal].

  * **minimalist**

    I want a minimal terminal emulator with just a terminal window and access to the shell while multiplexing handles a terminal multiplexor. Thus no packed ones like [Kitty][Kitty] or [WezTerm][WezTerm].

  * **performance**

    I want a responsive terminal emulator with offloaded rendering to the GPU for smoothness while staying lightweight. Thus no Electron-based ones like [Hyper] or [Tabby].
</details>
